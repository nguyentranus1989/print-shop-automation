  =====================|=============|==================
  Global pointer        | 0x176B98    | 0x1D2F10
  CTaskManager access   | appVt[22]   | +0x70
  AddFile vtable index  | vtable[7]   | vtable[9]
  File vector offset    | +0x28       | +0x08
  Display vector offset | +0x48       | +0x28
  CTaskInfo struct size | 0xB08       | 0xCA0
  Filename field offset | +0x5B0      | +0x140
  UI refresh message    | 0x7F4       | 0x7F4 (same)

CRITICAL SAFETY NOTES DOCUMENTED:
  1. AddFile PREPENDS (inserts at index 0)
     - Must diff file vector to find new entry

  2. DO NOT copy to print vector on UV
     - Different object type, causes R6025 pure virtual call crash

  3. Only copy to display vector (both DTF and UV)
     - Display vector and print vector are distinct types

  4. Dangerous vtable indices (UV only):
     - vt[6], vt[11], vt[12] cause immediate crash

DISCOVERY METHOD DOCUMENTED:
  - Python memory scanners (identify offset signatures)
  - TCC compiler (lightweight, no Ghidra required)
